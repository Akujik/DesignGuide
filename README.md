# DesignGuide
各个项目的设计规范做调取用
找出目标网站，最好下载几个具有代表性的网页 download 源代码文件，同时用 chrome dev cmd+shift+p 完整截屏。这是做元数据
放在分析文件夹中，调取 slash /design-analystic 这个命令
提示词
- 告诉它分析目标文件在哪里
- 辅助网页截屏让其进行比对
- 由于数据量较大，需要让其进行检测机制，随时记下来分析的结果，防止 ai 上下文会忘记遗漏
- 接着统计完以后，ai 会产生一个报告，将总结性的样式统计出来，那么这个时候只是原子形式的，所以要跟图片进行比对还有哪些地方忘记了
- 接着就开始一点点调整，一般网页比较直观，很多时候要有耐心将其沉淀下来
- 找到自己想要的组件，一点点抠下来。 chrome dev 定位 copy outhtml + copy css，加上图片一并喂给 ai，告诉他需要什么

## 防遗漏检查机制

### 分析前检查
1. 确认文件大小和行数
2. 如果文件>5000行，分段分析
3. 使用检查清单模板

### 分析中检查
1. 每个类别提取后立即统计
2. 关键词搜索验证(color, font, animation等)
3. 对比源码确认数量

### 分析后检查
1. 生成统计报告
2. 标注可能遗漏的部分
3. 提供补充分析建议

### 输出格式
必须包含:
- ✅ 已提取: XXX个
- ⚠️ 可能遗漏: 搜索发现还有XXX个未分析
- 📋 建议: 回到源码补充分析XXX部分
